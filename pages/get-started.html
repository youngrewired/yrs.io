Get started

This is a short guide and activity to help you understand the basics of building websites. We'll run through the simple HTML, CSS and Javascript, and build a small todo list website to put your skills to the test.

Part 1 - Learn HTML Basics

  1. Go to codepen.io and click 'New Pen'

  2. Type in the following in the HTML pane 

        Hello There
    Note: this displays correctly in the output, but there's no way of specifying what should be bold, italic, or underlined, etc. To do this, we use HTML tags. Add tags start with < and end in >. The content in between describes the tag.

  3. Replace the content with the following and note the output 

        <h1>Hello There
    Note: the <h1> tag stands for heading level 1, the largest of six. The others are <h2>, <h3>, <h4>, <h5> and <h6>.

  4. On the second line, write the following 

        <h1>Hello there
    <p> My name is {Your name}
    Note: <p> stands for paragraph, which is formatted as 'normal' text. The output should all be large, and that's because we need to tell the browser when a tag is finished. We do that using an end tag...

  5. Add end tags to your code 

        <h1>Hello there</h1>
    <p>My name is {Your name}</p>

  6. We're now going to learn about parent and child elements by creating a bulleted list. Think about the structure of a list - there is the list in it's entirety, and then individual bullet points. The tag for the list is <ul> and for each point, the <li> tag is used (it stands for list item). Add this code below the paragraph. 

        <ul>
        <li>Point 1</li>
        <li>Point 2</li>
        <li>Point 3</li>
    </ul>

  7. Now we'll insert an image. Before we start, we'll need a link to an image. You can use any direct image link - but here's the Young Rewired State logo link for use in this guide: http://www.yrs.io/static/images/yrs-logo-red.png

    Add this line to your code:

        <img src="http://www.yrs.io/static/images/yrs-logo-red.png">

    The image tag has two things of note. Firstly, an 'attribute'. These work very similarly to CSS property rules, which we will learn about later. Attributes give more information about your element. This is the structure of an element with an attribute tag:

        <element attribute="value" attribute2="value 2"></element>

    The second thing of note is the absence of a closing tag, like we have seen on elements so far. The <img> tag is one of very few which doesn't need a closing tag. Assume all do, and I'll point out when that is different.

Part 2 - Learn CSS Basics

We're now going to work in the CSS pane of CodePen. Before we do, please change the following setting: click the settings icon in the CSS pane, and choose 'Normalize' instead of 'Reset'.

CSS is structured like this:

    element {
        property: value;
        property2: value2;
    }

The element is what sits in between < and >, the property is what you want to set/change, and the value is what you're changing it to. For example:

    h1 {
        color: red;
    }

We're going to go through the CSS quite quickly, and learn new properties as we need them.

  1. Change the background, text colour and justification of the <h1> element. We'll be setting four properties in our CSS. Type the following: 

        h1 {
        background: red; 
        color: white;
        width: 100%;
        text-align: center;
    }

  2. That's some great styling to get started, and before we make any more changes, we're going to use a new property - padding. Padding adds some space inside of an element, now is a great time to show it off. Amend your code.

        h1 {
        background: red; 
        color: white;
        width: 100%;
        text-align: center;
        padding: 20px;
    }

  3. I'd like to note how you can target child elements inside CSS. You can use a space to seperate them, here the element on the left is the parent, and the one on the right is the child. Like so:

        ul li {

    }

  4. Now we're going to style the list. /* This is a CSS comment which will not affect the code. I will use them to explain each line. */ 

        ul li {
        background: lightblue;
        color: white;
        width: 80%; /* this will only span 80% of the width of the page */
        margin-left: 10%; /* this will push the element 10% on the left */
        padding: 20px; /* adds spacing inside the elements */
        margin-bottom: 10px; /* adds spacing below each element */
        border-radius: 5px; /* Rounds the edges of each item */
    }

If you want to take a look at more CSS properties - check out the Mozilla Developer Network reference.

The ToDo List Project

We'll be building a simple todo list application. We'll be using HTML, CSS, and Javascript (which we'll explore when we are using it).

Here's a reference for the finished project, along with the code in case you get stuck.

Before writing the HTML, it's important the think about the structure of a web page.

  1. Let's create and style the heading

        /* HTML */

    <header>
        <h1>ToDo List</h1>
    </header>



    /* CSS */

    header {
        background: #E34F50;
        color: white;
        padding: 10px;
        text-align: center;
        margin-bottom: 20px;
    }

    The <header> tag is meant to be used specifically when creating headers. It adds no extra styling by default (like, for example, the <li> adds bullet points), but makes the code easier to read and understand.

    The code #E34F50 refers to a colour. You can also use keywords use as 'red', or 'navy'. This specific code is YRS' brand colour.

  2. Now let's add the text box. We do this by using the <input> tag (which 'self closes' like the <img> tag). We then tell it what type of content to accept using the type="" attribute. Add this code:

        <input type="text" />

    Try this: change the word 'text' for 'date', 'password', or 'file' and see hwo the input box changes. Just make sure it is changed back to 'text' before continuing.

  3. Let's add the clickable explanation of the text box before continuing. Add this code above the input:

        <label>What would you like to remember?</label>

    To make it clickable, you need some way to link the label and the input. To do this, we have to add an attribute to both elements. Replace the input and label with the following code: 

        <label for="todo">What would you like to remember?</label>
    <input id="todo" type="text" />

    The for attribute refers the the <input> with the matching id.

  4. Let's style it! 

        label {
        display: block;
        text-align: center;
    }


    input[type="text"] {
        width: 40%;
        margin-left: 30%; 
        margin-top: 15px;
        font-size: 20px;
        padding: 10px;
        box-sizing: border-box;
        text-align: center;
        border-radius: 5px;
        border: 1px solid grey;
    }

    There's lots to talk about here.

    display: inline;

    The display property is one of the most important for controlling layout in CSS. Every element has a default (generally block or inline). The <label> tag by default has a value of inline.

    inline doesn't disrupt the flow of a document, but allows them to be identified. A link in a paragraph is a good example of this. block elements start on a new line and stretches out to the left and right as far as it can.

    In order to make the <label> sit on it's own line, we must change it's default display property.

    box-sizing: border-box

    The key concept here is quite complex, so don't worry if you don't fully understand it at this point. When you add padding to an element, you add some spacing to the inside of it. By default, this makes the element slightly bigger. Using box-sizing: border-box changes that default and keeps the original element sized correctly.

    border: 1px solid grey

    This is actually good CSS shorthand for three different rules:

        border-width: 1px;
    border-style: solid /*other options are dotted and dashed*/
    border-color: grey;
    Note: the border sits outside of the padding, but inside the margin. Don't worry about this, but if you want more information, search for 'css box model'

  5. Let's put in the go button for now, and we'll hook it up and make it work later. 

        /* HTML */

    <p class="go">Go</p>


    /* CSS */

    p.go {
        text-align: center;
        color: blue;
    }

    p:hover {
        color: darkblue;
    }

    Generally, we wouldn't use the <p> for a button, but this is a demonstration of how, if you want, the tags can be interchangeable at certain points.

    In CSS, a period (.) is used to represent the class attribute. As the <p> has a class attribute of the value go, you can use p.go in the CSS to refer to all paragraph tags with the specific class of 'go'.

    p:hover

    :hover is a 'psudoclass', and this one tells the browser to style an element in a certain way when it is being interacted with. As it's probably clear, this psudoclass changes the styling when the element is hovered over with the cursor pointer.

    Try this: change the <p> for a <button>, or a self-closing <a> tag. Don't forget to also change the CSS to target the right element!

  6. Let's create and style the list. 

        /* HTML */

    <ul>
        <li>Point 1</li>
        <li>Point 2</li>
        <li>Point 3</li>
    </ul>

     

    /* CSS */

    ul {
        -webkit-padding-start: 0;
    }


    ul li {
        list-style-type: none; /* Removes bullet points */
        width: 40%; 
        margin-left: 30%;
        background: #7EC7E3; /* YRS Brand Blue */
        color: white;
        padding: 10px;
        margin-bottom: 10px;
        box-sizing: border-box;
        border-radius: 5px;
    }

    -webkit-padding-start: 0; - let's talk about default browser styling. Browsers are awesome, they make our jobs as developers easy by looking at our HTML and deciding what type of styling is appropriate (bullet points on a list, or a blue underline on links). By default, all list have some spacing on the left, and it's this rule which overrides that.

Let's add some Javascript!

This thing we've built is great, but we need to write a small amount of 'logic' code which takes the contents of the field when a user clicks 'go' and adds it to the list.

  1. Click the settings icon in the JS pane in CodePen, and make sure that 'use the latest version of' is selecting jQuery.

  2. Type the following in the Javascript pane:

        $("p.go").click(function(){

    });

    We always start a new 'event listener' (a piece of code which watches for the user to interact) with the same general formatting.

  3. Add the middle two lines lines in to the function

        $("p.go").click(function(){
        var item = $("#todo").val();
        $("ul").prepend("<li>" + item + "</li>");
    });

    In the first line, note the var keyword. This allows you to store whatever is to the right of the equals sign to the word item. Now, whenever you want to type that code, all you have to do is type item instead.

    .val(); gets the value of an element. In this case, the item with the ID of todo (in CSS and Javascript, you can use # to represent the ID attribute, similar to the . for classes we looked at before). The #todo element is actually referring to the text input <input id="todo" type="text" />. We end each instruction with a semicolon (;).

    So now, when the go button is clicked, the value of the text box is stored in a variable called item.

    The second line starts by targeting the <ul> element with $("ul"). and then tells the browser what to do with it.

    We are going to add content inside the <ul>, but at the very start. To do this, we use the .prepend(), placing the content we want inserted inside of the brackets.

    Normally, we would use quotes around everything, but we aren't meant to for variables. In order to string this all together, we have the add the variable to the end of the opening tag, and then add the closing tag on the variable. It is the same as typing this: 

        $("ul").prepend("<li> This is a point </li>");

  4. Test everything works by typing in the box and clicking go.
